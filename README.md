Gestion des commits et des versions : 

- Chaque nouveau changement dans le code devra être concretisé par un commit sur la branche "dev-quality-system"
- Tous les commits doivent être spécifiés comme ceci : " Update : *Description des modifications/ajouts* "
- Concernant les changements de version, l'utilisateur devra lancer la commande suivante dans le terminal afin de modifier la version : .<b>\build.ps1 -version "*numero*.*de*.*version*"</b>
- Il y'a 3 numeros différents concernant la version : Le premier numéro doit être incrémenté en cas de modifications majeurs, celui du milieu en cas de modifications mineures et celui de la fin en cas de simples correctifs.
